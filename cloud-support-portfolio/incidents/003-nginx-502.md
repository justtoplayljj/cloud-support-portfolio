Incident 001 - Nginx 502 Bad Gateway
Overview
Date: 2026-06-01
Environment: Ubuntu 22.04
Service: Nginx + Java Application
Severity: Medium

Incident Description
用户访问业务系统时出现：
502 Bad Gateway
页面无法正常打开。

Symptoms
浏览器访问：
http://192.168.1.100
返回：
502 Bad Gateway
检查Nginx状态：
systemctl status nginx
结果：
active (running)
Nginx运行正常。

Investigation
检查Nginx错误日志：
tail -100 /var/log/nginx/error.log
发现：
connect() failed (111: Connection refused)
while connecting to upstream
说明Nginx无法连接后端应用。

检查监听端口：
ss -lntp | grep 8080
结果：
无输出
应用端口未监听。

检查Java进程：
ps -ef | grep java
结果：
无Java进程
应用服务异常退出。

Root Cause
开发人员发布新版本后配置错误。
导致Java应用启动失败。
Nginx无法连接后端服务。
最终返回502错误。

Resolution
恢复正确配置：
cp app.conf.bak app.conf
重新启动服务：
systemctl restart app
确认监听端口：
ss -lntp | grep 8080
结果：
LISTEN 0 128 0.0.0.0:8080
访问恢复正常。

Preventive Actions
1.上线前增加配置校验
2.增加应用存活监控
3.配置自动告警
4.建立回滚流程

Lessons Learned
Nginx 502问题不一定来自Nginx本身。
优先检查：
后端服务状态
端口监听
应用日志
配置变更记录
